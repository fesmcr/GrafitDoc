CMeasure

CMeasure​

The CMeasure class offers a suite of methods designed to compute different types of measurements for a graph.

Constructor​

Initializes a new instance of the CMeasure class for a given graph.

Parameters

  • Graph - The graph on which measurements will be performed. It must implement the IMeasurableGraph interface.

Example


var measurement = new CMeasure(graph);

UseDirectedEdges​

Specifies whether directed edges should be used in measurement results.

Remarks

Setting this property affects how edge-related measurements are calculated and interpreted. When set to true, edge-related measurements will be calculated for each direction separately. When false, the values for both directions of a single edge will be summed up.

Value

True if measurements should consider edge directionality; false otherwise. By default is true.

Example


var measurement = new CMeasure(graph);
measurement.UseDirectedEdges = true;

CalculateBC​

Calculates the betweenness centrality for both nodes (BCNodes) and edges (BCEdges) in the graph.

Parameters

  • ODWeightMatrix - The origin-destination weight matrix representing the weights of the shortest paths between all pairs of nodes.

  • GPU - A boolean indicating if GPU acceleration should be utilized. Set to true for GPU computation.

Remarks

Betweenness centrality measures the importance of nodes and edges based on their participation in the shortest paths across the graph.

The betweenness centrality (BC) for a node or edge, denoted as kkk, quantifies its significance by aggregating the weights of all shortest paths that traverse through kkk. Specifically, the BC value for kkk is determined by the formula:

BCk=∑i≠jODk[i][j]BC_k = \sum_{i \neq j} OD_k[i][j]BCk​=i=j∑​ODk​[i][j]

Here, ODk[i][j]OD_k[i][j]ODk​[i][j] represents the weight of the shortest path from node iii to node jjj that includes kkk.

Example

  • C#
  • Python
var measurement = new CMeasure(graph);
var GPU = true;
measurement.CalculateBC(ODWeightMatrix, GPU); 
var BCN = measurement.BCNodes;
var BCE = measurement.BCEdges;
measurement = CMeasure(Graph)
GPU = True
measurement.CalculateBC(Array[Array[float]](ODWeightMatrix), GPU)
BCN = measurement.BCNodes
BCE = measurement.BCEdges

CalculateBCGroup​

Calculates the betweenness centrality for edge groups, considering only paths with non-zero weights.

Parameters

  • ODWeightMatrix - The matrix indicating the presence (some non-zero weight) or absence (0) of the shortest paths between all pairs of nodes.

  • EdgeGroups - A list of edge groups, where each group contains the indices of edges considered together for betweenness centrality calculation.

Remarks

This method focuses on calculating the betweenness centrality for edges, assigning a weight of 1 for each considered path. The centrality value for a group of edges reflects the number of unique shortest paths that pass through at least one edge in the group. Paths corresponding to a weight of 0 in the ODWeightMatrix are excluded from the calculation.

BCgroup=∑i≠jσij(group)BC_{group} = \sum_{i \neq j} {\sigma_{ij}(group)}BCgroup​=i=j∑​σij​(group)

where σij(group)\sigma_{ij}(group)σij​(group) is the number of those paths passing through at least one edge in the group.

Returns

An array of betweenness centrality values for each group, where each value represents the count of unique paths passing through any at least one in the group.

Example

  • C#
  • Python
var measurement = new CMeasure(graph);   
var BCEdgeGroup = measurement.CalculateBCGroup(ODWeightMatrix, EdgeGroups);  
measurement = CMeasure(Graph)  
BCEdgeGroup = measurement.CalculateBCGroup(Array[Array[float]](ODWeightMatrix), List[List[int]](EdgeGroups))

CalculateBCWithAttractors​

Calculates betweenness centrality for edges (BCEdgesAttractors), emphasizing the influence of specified edge attractors within paths.

Parameters

  • ODWeightMatrix - The matrix indicating the presence (some non-zero weight) or absence (0) of the shortest paths between all pairs of nodes.

  • EdgeAttractorWeights - The attractor weights for each edge, quantifying their relative influence or attractiveness.

Remarks

This method enhances the concept of betweenness centrality by incorporating the notion of 'attractors'—specific qualities or weights assigned to edges that make them more 'attractive' or influential in the network's flow. The process unfolds in two main steps:

  1. Total Attractor Weight on a Path: The method sums up the attractor weights of all edges along each path.

  2. Proportion of Individual Attractor Weight: The centrality contribution of each edge is determined by its attractor weight relative to the total attractor weight of the path it's on. Specifically, for an edge with attractor weight AAA on a path with total attractor weight TTT, its contribution to the path's centrality is proportionate to AT\frac{A}{T}TA​ multiplied to the ODWeightMatrix weight of a path.

The following formula is used:

BCa=∑i≠jODa[i][j]×(EdgeAttractorWeights[a]PathTotalAttractorWeight)BC_a = \sum_{i \neq j} OD_a[i][j] \times \left(\frac{EdgeAttractorWeights[a]}{PathTotalAttractorWeight}\right)BCa​=i=j∑​ODa​[i][j]×(PathTotalAttractorWeightEdgeAttractorWeights[a]​)

In this formula, ODa[i][j] OD_a[i][j]ODa​[i][j] denotes the weight of shortest path from node iii to node jjj that includes attractor edge aaa. The PathTotalAttractorWeightPathTotalAttractorWeightPathTotalAttractorWeight denotes the sum of edge weights in a path from node iii to node jjj as in the list EdgeAttractorWeights.

Example

  • C#
  • Python
var measurement = new CMeasure(graph); 
measurement.CalculateBCWithAttractors(ODWeightMatrix, EdgeAttractorWeightsList);
var BCEdgesAttractors = measurement.BCEdgesAttractors;
  
measurement = CMeasure(Graph)  
measurement.CalculateBCWithAttractors(Array[Array[float]](ODWeightMatrix), List[float](EdgeAttractorWeightsList))
BCEdgesAttractors = measurement.BCEdgesAttractors

CalculateBCVisibility​

Calculates the visibility betweenness centrality, quantifying the network flow visibility from specified objects.

Parameters

  • ODWeightMatrix - The matrix indicating the presence (some non-zero weight) or absence (0) of the shortest paths in calculation.

  • RaysHitToObjects - A flat array linking each vantage point's ray to its visible object ID (or -1 if ray hits nothing).

  • EdgeToVantagePoints - A dictionary associating edges with their corresponding vantage points.

  • ObjectsCount - The number of objects included in the visibility analysis.

Remarks

This method evaluates how visible objects are to the paths within the network, effectively measuring how many unique paths have a view of each object.

The array RaysHitToObjects effectively maps the visibility rays emitted from each vantage point to the objects they can see.
Each edge can have many vantage points. The sum of number of vantage points per each edge in the EdgeToVantagePoints is a total number of vantage points. So the following is true:

int raysCountPerVantagePoint = RaysHitToObjects.Length / TotalNumberOfVantagePoints;

Visibility betweenness centrality for an object is calculated using the formula:

BCobject=∑i≠jODvisibleEdge[i][j]BC_{object} = \sum_{i \neq j} OD_{visibleEdge}[i][j]BCobject​=i=j∑​ODvisibleEdge​[i][j]

where ODvisibleEdge[i][j]OD_{visibleEdge}[i][j]ODvisibleEdge​[i][j] is a weight of the shortest path from node iii to node jjj that have at least one edge from which the objectobjectobject is visible.

Returns

An array where each value represents the total visibility (summed OD weights) of network paths from an object's perspective.

Example

  • C#
  • Python
var measurement = new CMeasure(graph); 
var BCVisibility = measurement.CalculateBCVisibility(ODWeightMatrix, RaysHitToObjects, EdgeToVantagePoints, ObjectsCount); 
  
measurement = CMeasure(Graph)   
BCVisibility = measurement.CalculateBCVisibility(Array[Array[float]](ODWeightMatrix), Array[int](RaysHitToObjects), Dictionary[int, List[int]](EdgeToVantagePoints), ObjectsCount)

CalculateCC​

Calculates the closeness centrality for both nodes (CCNodes) and edges (CCEdges) based on the sum of shortest path distances from a specified weight matrix.

Parameters

  • ODWeightMatrix - The origin-destination matrix indicating the presence (some non-zero weight) or absence (0) of the shortest paths in calculation.

  • WeightMatrixId - The identifier of the weight matrix to use for distances retrieving. Default is 0.

  • GPU - A boolean indicating if GPU acceleration should be utilized. Set to true for GPU computation.

Remarks

This method diverges from the classical approach to calculating closeness centrality (CC). Instead of averaging the inverse of the distances, it directly sums the distances of the shortest paths as provided by the specified weight matrix. This approach highlights nodes and edges with lower aggregate distances across the network, potentially offering a more intuitive measure of centrality in certain contexts.

The closeness centrality for a node iii is calculated using the formula:

CCi=∑i≠jdist(i,j)×OD[i][j]CC_i = \sum_{i \neq j} dist(i, j) \times OD[i][j] CCi​=i=j∑​dist(i,j)×OD[i][j]

Where dist(i,j)dist(i, j)dist(i,j) denotes the shortest path distance between nodes iii and jjj according to input WeightMatrixId and OD[i][j]OD[i][j]OD[i][j] the weight of this path. For edges, depending on the UseDirectedEdges parameter, the method identifies the initial directed or undirected edge in the path from node iii and accumulates the path distance to that edge ID.

Example

  • C#
  • Python
  var measurement = new CMeasure(graph);
  var GPU = true;
  measurement.CalculateCC(ODWeightMatrix, WeightMatrixId, GPU); 
  CCN = measurement.CCNodes;
  CCE = measurement.CCEdges; 
measurement = CMeasure(Graph)
GPU = True
measurement.CalculateCC(Array[Array[float]](ODWeightMatrix), WeightMatrixId, GPU)
CCN = measurement.CCNodes
CCE = measurement.CCEdges

CalculateCCNodesEdgesAverageCPU​

Calculates closeness centrality for nodes and computes an average-based closeness centrality for edges.

Parameters

  • ODWeightMatrix - The origin-destination matrix indicating the presence (some non-zero weight) or absence (0) of the shortest paths in calculation.

  • WeightMatrixId - The identifier of the weight matrix to use for distances retrieving. Default is 0.

Remarks

This method calculates the closeness centrality (CC) for nodes, using the sum of distances from the ODWeightMatrix. For edges, instead of summing path distances directly, it computes an average closeness centrality value based on the closeness centrality of the nodes it connects.

The formula for calculating the closeness centrality of a node:

CCnode=∑i≠jdist(i,j)×OD[i][j]CC_{node} = \sum_{i \neq j} dist(i, j) \times OD[i][j] CCnode​=i=j∑​dist(i,j)×OD[i][j]

Where dist(i,j)dist(i, j)dist(i,j) denotes the shortest path distance between nodes iii and jjj and OD[i][j]OD[i][j]OD[i][j] the weight of this path. For an edge connecting nodes iii and jjj, the closeness centrality is derived as:

CCedge=CCnodei+CCnodej2CC_{edge} = \frac{CC_{node_i} + CC_{node_j}}{2}CCedge​=2CCnodei​​+CCnodej​​​

where CCnodeiCC_{node_i}CCnodei​​ and CCnodejCC_{node_j}CCnodej​​ are the closeness centrality values of the nodes at either end of the edge.

Example

  • C#
  • Python
  var measurement = new CMeasure(graph); 
  measurement.CalculateCCNodesEdgesAverageCPU(ODWeightMatrix, WeightMatrixId); 
  CCN = measurement.CCNodes;
  CCE = measurement.CCEdges; 
measurement = CMeasure(Graph) 
measurement.CalculateCCNodesEdgesAverageCPU(Array[Array[float]](ODWeightMatrix), WeightMatrixId)
CCN = measurement.CCNodes
CCE = measurement.CCEdges

CalculateCCNodesEdgesSumCPU​

Calculates closeness centrality for nodes and edges, with edges centrality based on a detailed path distance calculation.

Parameters

  • ODWeightMatrix - The origin-destination matrix indicating the presence (some non-zero weight) or absence (0) of the shortest paths in calculation.

  • WeightMatrixId - The identifier of the weight matrix to use for distances retrieving. Default is 0.

Remarks

For nodes, closeness centrality is determined by the sum of shortest path distances according to input WeightMatrixId multiplied by the corresponding weight from the ODWeightMatrix (CCNodesDistanceSum). For edges, closeness centrality involves a calculation that considers the path distance between edges, starting from the middle of an edge to another edge's middle. The process includes:

  1. Identifying the nodes at the ends of the origin (o1o1o1, o2o2o2) and destination edges (d1d1d1, d2d2d2).
  2. Calculating the shortest path distance from the end of the origin edge (o2o2o2) to the beginning of the target edge (d1d1d1).
  3. Adding half the distance of each edge to simulate starting from or arriving at the edge's midpoint.
  4. Multiplying obtained distance by the corresponding weight from the ODWeightMatrix.

Edge closeness centrality formula is the following (CCEdgesDistanceSum):

CCedgei=∑i≠jOD[o2][d1]×(dist(o2,d1)+dist(o1,o2)2+dist(d1,d2)2)CC_{edge_i} = \sum_{i \neq j} OD[o2][d1] \times (dist(o2, d1) + \frac{dist(o1, o2)}{2} + \frac{dist(d1, d2)}{2})CCedgei​​=i=j∑​OD[o2][d1]×(dist(o2,d1)+2dist(o1,o2)​+2dist(d1,d2)​)

Where edgeiedge_iedgei​ as origin has nodes o1o1o1 and o2o2o2. And nodes d1d1d1 and d2d2d2 belong to an edgejedge_jedgej​ as destination.

This method also tracks the count of paths passing through each node (CCNodesPathsCount) and edge (CCEdgesPathsCount), enabling calculations of both classical and customized closeness centrality metrics.

Example

  • C#
  • Python
  var measurement = new CMeasure(graph); 
  measurement.CalculateCCNodesEdgesSumCPU(ODWeightMatrix, WeightMatrixId); 
  CCN = measurement.CCNodes;
  CCE = measurement.CCEdges; 
  
 var CCED = measurement.CCEdgesDistanceSum; 
 var CCEP = measurement.CCEdgesPathsCount;
 var CCND = measurement.CCNodesDistanceSum;
 var CCNP = measurement.CCNodesPathsCount;
measurement = CMeasure(Graph) 
measurement.CalculateCCNodesEdgesSumCPU(Array[Array[float]](ODWeightMatrix), WeightMatrixId)
CCN = measurement.CCNodes
CCE = measurement.CCEdges

CCEP = measurement.CCEdgesPathsCount
