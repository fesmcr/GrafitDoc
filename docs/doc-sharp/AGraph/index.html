AGraph

AGraph​

Represents an abstract base class for a graph data structure.

Nodes​

Gets the list of nodes in the graph.

Value

The list of objects representing the nodes within the graph.

NoValue​

Gets the value used to represent an absence of value or an invalid state.

Value

An integer indicating the absence of a valid value. By default set to -1.

UndirectedEdges​

Gets the collection of undirected edges in the graph.

Value

A list of integer arrays, where each array contains two elements representing the node IDs.

Example


var undirectedEdges = graph.UndirectedEdges;
foreach (var edge in undirectedEdges)
{
    Console.WriteLine($"Edge between node ID {edge[0]} and node ID {edge[1]}");
}

NextMatrix​

Gets the 'Successor' matrix used to reconstruct the paths between nodes in the graph.

Value

A two-dimensional array of integers where each element [i][j] represents the next node on the shortest path from node i to node j.

AdjacencyMatrix​

Gets the adjacency matrix representing the graph.

Value

A two-dimensional array where each element [i][j] represents the presence (Edge Id) or absence (int.MaxValue) of an edge between node i and node j.

NodesCount​

Gets the total number of nodes in the graph.

Value

The count of nodes currently present in the graph.

WeightMatricesCount​

Gets the total number of weight matrices in the graph.

Value

The count of weight matrices that have been added to the graph.

IsDirected​

Indicates whether the graph is directed.

Value

True if the graph is directed, meaning that each segment in a network converted to 2 edges of opposite directions; false if the graph is undirected.

DirectedEdgesCount​

Gets the total count of directed edges in the graph.

Value

The count of directed edges, which is twice the number of undirected edges since each undirected edge is represented by two directed edges in opposite directions.

UndirectedEdgesCount​

Gets the total count of undirected edges in the graph.

Value

The number of undirected edges.

GetShortestPathDistance​

Retrieves the shortest path distance between two nodes using a specified weight matrix.

Parameters

  • FromNodeId - The ID of the starting node.

  • ToNodeId - The ID of the destination node.

  • WeightMatrixId - The identifier for the weight matrix to be used for lookup.

Remarks

This method directly accesses the specified weight matrix to retrieve the distance between the 'From' and 'To' nodes. If the 'From' and 'To' nodes are the same, it returns a default distance value defined by TZero, which represents the absence of distance or a path of length zero.

Ensure that the nodes and weight matrix ID are valid before calling this method to avoid out-of-range errors. This method assumes that the shortest path distances have been computed and stored in the weight matrices. See APShortestPathsByOneWeight and APShortestPathsByManyWeights

Returns

The shortest path distance of type (double or float) between the 'From' and 'To' nodes. Returns 0 if the 'From' and 'To' nodes are the same.

GetShortestPathNodeIds​

Retrieves the shortest path as a list of node IDs between two specified node Ids.

Parameters

  • From - The ID of the starting node (always a Primal one).

  • To - The ID of the destination node (always a Primal one).

Remarks

This method returns the list of node IDs that form the shortest path from a starting node to a destination node. Ensure that node IDs correspond to the primal graph (points), not to the dual graph nodes (lines). This method assumes that the shortest path distances have been computed and stored. See APShortestPathsByOneWeight and APShortestPathsByManyWeights.

If the starting and ending nodes are the same, the method returns a list containing only the starting node ID.

If a path cannot be found due to a broken path or if any intermediate node leads to a cycle the method returns null.

Returns

A list of node IDs representing the shortest path from the starting node to the destination node, or null if no valid path exists.

GetShortestPathNodeAndEdgeIds​

Retrieves the shortest path between two nodes as both node and edge IDs, considering the graph's directionality.

Parameters

  • FromNodeId - The ID of the starting node.

  • ToNodeId - The ID of the destination node.

  • IsDirected - Indicates whether the edge IDs list will contain directed or undirected edge IDs.

Remarks

This method first determines the shortest path by node IDs from the starting node to the destination node using GetShortestPathNodeIds. It then translates this path into edge IDs, which represent the connections between consecutive nodes in the path. The method return edges result as directed or undirected edge ids, using the appropriate function (GetDirectedEdgeIdByNodes or GetUndirectedEdgeIdByNodes) to find edge IDs based on node pairs. Ensure that node IDs correspond to the primal graph (points), not to the dual graph nodes (lines). If the starting and ending nodes are the same, the method returns a node list containing only the starting node ID and empty edge list.

Returns

A tuple containing two lists: the first list contains the node IDs in the shortest path, and the second list contains the corresponding edge IDs.

Example


var (nodeIds, edgeIds) = graph.GetShortestPathNodeAndEdgeIds(fromNodeId, toNodeId, isDirected);

GetShortestPathNodeAndEdgeIds​

Retrieves the shortest path between two nodes as both node and edge IDs, considering the graph's directionality.

Parameters

  • FromNodeId - The ID of the starting node.

  • ToNodeId - The ID of the destination node.

  • IsDirected - Indicates whether the edge IDs list will contain directed or undirected edge IDs.

  • NodePathIds - An output parameter to hold the list of node IDs comprising the shortest path.

  • EdgePathIds - An output parameter to hold the list of edge IDs comprising the shortest path.

Remarks

This method is the same as GetShortestPathNodeAndEdgeIds but result returned via out parameters.

Example


graph.GetShortestPathNodeAndEdgeIds(fromNodeId, toNodeId, isDirected, out List<int> nodePath, out List<int> edgePath);

GetAngleBetweenEdgesByNodesIds​

Calculates the angle between two edges defined by three node IDs.

Parameters

  • NodeId1 - The ID of the first node, forming the start of the first edge.

  • NodeId2 - The ID of the second node, shared by both edges.

  • NodeId3 - The ID of the third node, forming the end of the second edge.

Remarks

This method calculates the angle between two edges that share a common node. The edges are defined by the nodes (NodeId1, NodeId2) and (NodeId2, NodeId3). It uses the geometric positions of these nodes to compute the angle.

Returns

The absolute value of the angle (in degrees) between the two edges formed by the three nodes.

GetUndirectedEdgesAsNodeIdsFlatArray​

Constructs a flat array representing undirected edges in a graph. Each pair of consecutive elements in the array represents an edge, where the first element of the pair is the ID of the first node, and the second element is the ID of the second node of the edge.

Remarks

The array is structured so that for an edge at index i in the list of edges, the elements at positions 2i and 2i + 1 in the flat array correspond to the IDs of the first and second nodes of that edge, respectively.

Returns

A flat array of integers where each successive pair of elements represents the node IDs of an edge.

Example


var flatEdges = graph.GetUndirectedEdgesAsNodeIdsFlatArray();
// If the graph contains an edge between nodes 1 and 2, and another edge between nodes 3 and 4,
// the flatEdges array might look like [1, 2, 3, 4], assuming these are the only edges in the graph.

JoinForwardAndBackwardWeightsAsFlatArray​

Combines the forward and backward weight arrays of a specified weight matrix into a single, flat array.

Parameters

  • WeightMatrixId - The identifier of the weight matrix whose forward and backward weights are to be joined. This ID should correspond to an existing weight matrix within the graph.

Remarks

This method is designed to work with both directed and undirected graphs. For directed graphs, it combines the weights of edges in both forward and backward directions, creating a flat array that represents concatenated the forward and backward weights. For undirected graphs, it simply returns the forward weights.

GetWeightMatrixById​

Retrieves a weight matrix associated with a given identifier.

Parameters

  • Id - The identifier for the weight matrix to retrieve.

Remarks

This method assumes that a collection of weight matrices is indexed by identifiers, equal to the order of adding the matrices to the graph via AddWeightMatrix function.

Returns

A 2-dimensional array of type (float or double), representing the weight matrix for the graph associated with the given ID.

IsAdjacent​

Determines whether two nodes in the graph are adjacent.

Parameters

  • Node1Id - The identifier of the first node.

  • Node2Id - The identifier of the second node.

Remarks

This method checks the adjacency matrix of the graph to determine if there is a direct edge between two nodes.

Returns

true if there is a direct edge between the two nodes; otherwise, false.

IsNodeIdValid​

Checks if a given node ID is valid within the current graph.

Parameters

  • NodeId - The ID of the node to validate.

Remarks

This method verifies that the provided node ID is within the valid range of nodes in the graph, which is determined by the current count of nodes (NodesCount).

Returns

true if the node ID is less than the current count of nodes in the graph; otherwise, false.

GetUndirectedEdgeId​

Returns an udirected edge ID.

Parameters

  • EdgeId - The ID of the edge.

Remarks

If EdgeId is a directed edge ID, then a corresponding undirected edge ID is returned. In case the EdgeId corresponds to already undirected edge, it will be returned as it is.

Returns

The undirected edge ID.

GetDirectedAndUndirectedEdgeIdByNodes​

Retrieves the directed and undirected edge IDs for a pair of nodes.

Parameters

  • Node1Id - The ID of the first node in the pair.

  • Node2Id - The ID of the second node in the pair.

Remarks

This method first retrieves the directed edge ID for the node pair specified by Node1Id and Node2Id using the GetDirectedEdgeIdByNodes method. It then obtains the corresponding undirected edge ID by converting the directed edge ID via the GetUndirectedEdgeId method. In case nodes are not adjacent int.MaxValue will be returned.

Returns

A tuple containing the directed edge ID and the undirected edge ID for the specified node pair.

Example


(int directedEdgeId, int undirectedEdgeId) = graph.GetDirectedAndUndirectedEdgeIdByNodes(node1Id, node2Id);

GetUndirectedEdgeIdByNodes​

Retrieves the undirected edge ID for an edge defined by two node IDs.

Parameters

  • Node1Id - The ID of the first node.

  • Node2Id - The ID of the second node.

Remarks

This method retrieves, first, the directed edge ID between the two specified nodes using GetDirectedEdgeIdByNodes and then converts it to an undirected edge ID using GetUndirectedEdgeId. In case nodes are not adjacent int.MaxValue will be returned.

Returns

The undirected edge ID corresponding to the edge between the specified nodes.

Parameters

  • Node2Id - The ID of the second node.

Remarks

This method retrieves the directed edge ID between the two specified nodes using In case nodes are not adjacent int.MaxValue will be returned.

GetOppositeEdgeId​

Retrieves the ID of the edge that is in the opposite direction to the given edge ID.

Parameters

GetNodeIdsByEdgeId​

GetSubgraphs​

Retrieves all subgraphs within the graph as a list of lists, where each list contains the node IDs of a subgraph.

Remarks

This method identifies and collects subgraphs by performing a depth-first search (DFS) starting from each unvisited node.

Returns

A list of subgraphs, each represented as a list of node IDs indicating the nodes that form the subgraph.

Example


List<List<int>> subgraphs = graph.GetSubgraphs();
foreach (var subgraph in subgraphs)
{
    Console.WriteLine($"Subgraph: {string.Join(", ", subgraph)}");
}

APShortestPathsByOneWeight​

Calculates all-pairs shortest paths in the graph using the Floyd-Warshall algorithm, based on a specified weight matrix and a tolerance for distance comparison.

Parameters

  • WeightMatrixId - The identifier of the weight matrix to use for calculating distances with the Floyd-Warshall algorithm.

  • GPU - A boolean indicating if GPU acceleration should be utilized. Set to true for GPU computation.

  • Tolerance - The tolerance within which two distances are considered equal. The tolerance is of type (double or float).

Remarks

This method employs the Floyd-Warshall algorithm to compute the shortest paths between all pairs of nodes in the graph, considering a single weight for each edge as determined by the WeightMatrixId. The Tolerance parameter is used to treat distances as equal if they fall within this specified tolerance.

The algorithm is executed on either the CPU or GPU, based on the GPU parameter. When true, GPU acceleration is enabled.

APShortestPathsByManyWeights​

Calculates all-pairs shortest paths in the graph using the Floyd-Warshall algorithm, based on 2 specified weight matrices and a tolerance for distance comparison.

Parameters

  • WeightMatrixId - The identifier of the weight matrix to use for calculating distances with the Floyd-Warshall algorithm.

  • RelativeMatrixId - Identifier for an alternative weight matrix to resolve cases when shortest paths by main weight matrix are equal within the tolerance.

  • GPU - A boolean indicating if GPU acceleration should be utilized. Set to true for GPU computation.

  • Tolerance - The tolerance within which two distances are considered equal. The tolerance is of type (double or float).

Remarks

This method employs the Floyd-Warshall algorithm to compute the shortest paths between all pairs of nodes in the graph, considering two weights for each edge as determined by the WeightMatrixId and RelativeMatrixId. If the shortest path between a pair of nodes is equal within the margin defined by the Tolerance parameter, the path with the smaller distance, as determined by the RelativeMatrixId, will be chosen as the solution.

The algorithm is executed on either the CPU or GPU, based on the GPU parameter. When true, GPU acceleration is enabled.

InitializeNextMatrix​

Initializes the 'Successor' matrix used for pathfinding, setting default values for direct paths and marking self-paths as invalid.

Remarks

This method sets up the 'Successor' matrix which is used for reconstructing paths after running pathfinding algorithms. Each cell in the matrix indicates the next node on the shortest path from origin node to destination node. Self-paths, or paths from a node to itself, are marked as invalid with a default value of -1 ( NoValue), indicating that such paths should not be considered in path reconstruction.

AddEmptyWeightMatrix​

Adds new edges weights and a corresponding weight matrix to the graph, initializing all weights.

Parameters

  • DiagonalValue - The weight value for paths from a node to itself within the matrix, indicating the self-path cost. By default double.MaxValue.

Remarks

This method creates a new weight matrix where each edge's weight to itself is set to the given DiagonalValue, by default double.MaxValue, indicating invalid travel from a node to itself. All other weights are initialized to represent infinite distance (double.MaxValue).

Following initialization, the method creates a list of undirected edge weights (forward), setting all to represent infinite distance. For directed graphs, a separate list for opposite direction edge weights (backward) is also created, applying the same infinite distance initialization. Note, that empty weight matrix can be added only if graph already have nodes and edges and weighted by at least one matrix.

Returns

The index of the newly added weight matrix, which can be used to reference it in future operations.

RemoveWeightMatrix​

Removes a specified weight matrix and its associated edge weights from the graph.

Parameters

  • WeightMatrixId - The identifier of the weight matrix to be removed.

Remarks

The method removes the weight matrix along with the associated undirected (forward) edge weights. For directed graphs, edges in opposite direction (backward) weights corresponding to the same weight matrix ID are also removed.

AddWeightMatrix​

Adds a new weight matrix to the graph and updates corresponding edge weights.

Parameters

  • WeightMatrix - The weight matrix to be added, representing the weights between nodes in the graph.

Remarks

This method adds the specified weight matrix to the graph. For directed graphs, both directions of edge weights (forward and backward) are created based on the input weight matrix. For undirected graphs, only undirected edge weights (forward) are created.

Returns

The identifier of the newly added weight matrix.

SetWeightMatrixValue​

Sets the weight value for an edge in a specified weight matrix.

Parameters

  • Value - The weight value to set for the edge.

  • FromNodeId - The ID of the source node of the edge.

  • ToNodeId - The ID of the destination node of the edge.

  • WeightMatrixId - The identifier of the weight matrix where the weight value is to be set.

Remarks

This method updates the weight of the edge from the origin node to the destination node in the specified weight matrix. Note that it updates only matrix without updating the edges lists. This is helpful for dual graphs only. In other cases the SetWeightMatrixAndEdgeValue should be used.

SetWeightMatrixAndEdgeValue​

Sets the weight for a specific edge in both the weight matrix and the corresponding edge list.

Parameters

  • Value - The weight value to set for the edge.

  • FromNodeId - The ID of the source node of the edge.

  • ToNodeId - The ID of the destination node of the edge.

  • WeightMatrixId - The identifier of the weight matrix where the weight value is to be set.

Remarks

Updates the specified edge's weight in the weight matrix and then adjusts the corresponding edge's value in the edge list. This ensures consistency between the graph's internal representations of edge weights.

FillWeightMatrixBySPRestore​

Fills weight matrices with distances of shortest paths, reconstructed from the 'Successor' matrix, using specified weight matrix IDs.

Parameters

  • WeightMatrixIds - An array of identifiers for the weight matrices to be updated with shortest path distances.

  • GPU - A boolean indicating if GPU acceleration should be utilized. Set to true for GPU computation.

Remarks

This method utilizes the 'Successor' matrix, which is calculated in previous steps, to retrace the shortest paths between all pairs of nodes and measure the existed paths according to other weights specified in parameter WeightMatrixIds.

Depending on the GPU parameter, this process is performed either on the CPU or the GPU.
